# 判断記録

## 2026-01-28 Issue #9: PR トリガー判定の厳格化

1. 判断内容の要約
   - 必須のステータスチェック対象は、`pull_request` / `pull_request_target` が常時発火すると判断できるワークフローのみに限定した。
2. 検討した代替案
   - `Reply to review` を名前パターンで除外する。
   - `pull_request` を含むワークフローは従来通りすべて対象にする。
3. 採用しなかった案とその理由
   - 名前パターン除外は特定ワークフローに依存し、同種の問題を再発しやすいため採用しなかった。
   - 従来通りすべて対象にすると、常時動作しないワークフローが必須チェックに残るため採用しなかった。
4. 前提条件・仮定・不確実性
   - 仮定: `types` を指定しない、または `synchronize` を含む PR トリガーが必須チェックに適する。
   - 不確実性: ジョブ内の `if` 条件や `paths` / `paths-ignore` による条件分岐は判定に含めていないため、常時実行されないケースが残る可能性がある。
5. 他エージェントによるレビュー可否
   - 可能。必要に応じて Gemini CLI / Claude Code で検証する。
